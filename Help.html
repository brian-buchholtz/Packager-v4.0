VMware Packager Help

Overview

VMware Packager is a tool to help simplify the building of PKGs and DMGs. Packager supports the following functionality:

  • Package signing
  • File ownership and permissions
  • Autogeneration of Munki plist

Projects

Projects are files containing metadata that describe how packages are built. They store essential information about the package itself, ownership, permissions and signing settings.

New Project
Creates a new empty project.
Open Project
Opens an existing project.
Save Project
Saves current project. It is possible to build a package, without saving project.

Project Settings

Project settings are defined on the main Packager user interface.

Name
This represents the app name.
Example: "Sample App"
Version
This represents the app version. VMware Workspace ONE UEM uses this value to track versioning.
Example: "1.0"
Identifer
This represents the unique app identifier or bundle ID.
Example: "com.company.Sample-App"
Root
This represents the starting point in the filesystem where the package is installed.
Example: "/"
Source
This represents the payload folder of the package.
Owner
This represents the owner of the installed package. Permissions can be set for the owner. For DMG creation, this is automatically handled. For PKG creation, the post installation script will have to be modified.
Group
This represents the group of the installed package. Permissions can be set for the group. For DMG creation, this is automatically handled. For PKG creation, the post installation script will have to be modified.
Everyone
This represents the permissions for everyone. For DMG creation, this is automatically handled. For PKG creation, the post installation script will have to be modified.
Sign
This represents which certificate will be used for package signing. This option has to be checked, if you want the package to be signed.
Target
This represents the location where the package and plist will be built.
